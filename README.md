# obj-task-1

Выполнен на ASP.NET WebApi. Запросы прописаны в `SubscribtionController`

предварительно в `appsettings.json` заполнить раздел `Smtp` - для работы рассылки сообщений.

сборка: `docker build -t apartmentspriceapi .`

запуск: `docker run -p 8080:8080 apartmentspriceapi`

При запуске создается база SQLite, применяются миграции.

#Описание API

`GET /api/subscription/` - Получение списка всех ссылок на квартиры и цены на них

`POST /api/subscription/` - Подписка на квартиру. 

Формат тела запроса: `{"ApartmentUrl": "", "UserEmail" : ""}`

#Описание сервисов

**ApartmentPrice** - достает из ссылки ID квартиры, отправляет запрос на API сайта, извлекает цену из ответа

**EmailNotification** - принимает в себя url квартиры, почту получателя, старую и новую цену. Отправляет шаблонное письмо получателю с данными об изменении цены квартиры.

**PriceChecker** - Сервис проверки цен квартир находящихся в БД. Использует ApartmentPrice и EmailNotification.

**PriceMonitor** - BackgroundService, раз в час запускает PriceChecker.

**SubscriptionService** - Управление подписками, создание и получение.

# Структура БД

В БД 3 таблицы:
1. Apartments - содержит информацию о цене квартиры
2. Subscribers - информация о подписчиках
3. Subscriptions - промежуточная таблица, с информацией конкретно о подписках, FK на Apartments и Subscribers

Так как у нас может быть как на одну квартиру подписано несколько разных почт, так и одна почта подписана на несколько разных квартир, то была выбранна именно такая архитектура, а не единая таблица, во избежание дублирования значений.

# Недостатки и улучшения

1. Отсутсвие валидации модели - сейчас пользователь может вставить любую инфу в запрос создания подписки, можно валидировать на входе через регулярки например
2. Потенциальный Timeout - допустим квартир станет много, тогда при проверке цен можно словить разворот. Можно кидать запросы с какой-то задержкой
3. Добавить метод получения подписок по конкретному email
4. Рассылка, вместо отправки по одному сообщению - сейчас сервис проверки цен высылает по одному письму на каждую подписку при изменении цены. Его можно доработать, чтобы он ориентировался на квартиры, а не на подписки, и рассылал сообщение.
5. Потенциальные ошибки в парсинге ответа api сайта - во время работы заметил, что некоторые квартиры цену возвращают формата string, а другие - числом. Причем названия полей тоже различаются. Можно ещё провести тестирование API для получения информации о возможных форматах, дабы ApartmentPrice не падал.
